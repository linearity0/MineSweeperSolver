# Minesweeper Auto Solver

自动扫雷器，基于三层递进策略自动完成经典扫雷游戏。采用确定性推理与蒙特卡洛采样相结合的方法，覆盖单点规则匹配、约束方程组推导和概率评估三种求解策略。

## 核心算法

### 游戏引擎

基于 BFS 的泛洪填充算法实现空格自动扩散：当翻开一个空白格（周围无雷）时，使用 `collections.deque` 做广度优先搜索，自动掀开所有相邻的安全区域。

**应用知识点**：BFS、队列、二维数组

### 第一层：单点推理（Single Point Deduction）

遍历每个已翻开数字格，利用集合运算和直接规则做出确定性判断：

- **规则 1**：如果 `剩余雷数 == 未翻开邻居数`，则所有邻居都是雷，全部标记
- **规则 2**：如果 `剩余雷数 == 0`，则所有邻居都安全，全部掀开

**应用知识点**：二维数组遍历、集合运算、邻域搜索

### 第二层：约束方程组推理（Constraint Propagation）

当单点推理无法推进时，将多个数字格的信息联合建模为约束方程组：

```
格子 A（数字 1，邻居 {X, Y}）     → 约束: {X, Y} = 1
格子 B（数字 2，邻居 {X, Y, Z}）  → 约束: {X, Y, Z} = 2
```

**核心算法——约束子集减法**：若约束 C₁ 的 cells 是约束 C₂ 的 cells 的子集，则可以通过 C₂ - C₁ 推导出新约束：

```
{X, Y, Z} - {X, Y} = {Z},  2 - 1 = 1  →  {Z} = 1  →  Z 是雷
```

推导出的新约束可能再次与其他约束产生子集关系，形成链式传播。该过程不断迭代直至收敛。

**应用知识点**：集合包含关系判断、约束传播、等式消元

### 第三层：概率评估（Monte Carlo Sampling）

前两层均无法推进时，意味着当前局面不存在确定性结论，需要进行概率猜测。采用蒙特卡洛方法：

1. 从已翻开格构建约束集
2. 随机采样大量满足所有约束的雷区配置（每次采样通过迭代修正逼近可行解）
3. 统计每个未翻开格子被分配为雷的频率
4. 选择含雷概率最低的格子点击

**应用知识点**：蒙特卡洛采样、概率模型、约束满足

## 运行指南

### 环境要求

- Python 3.8+
- pygame >= 2.5.0

### 安装与运行

```bash
# 安装依赖
pip install -r requirements.txt

# 启动 GUI
python src/main.py
```

### 操作说明

| 操作 | 方式 |
|------|------|
| 掀开格子 | 左键点击 |
| 标记/取消标记 | 右键点击 |
| 单步推理 | 点击「单步」按钮 或 按 `空格` |
| 自动求解 | 点击「自动求解」按钮 或 按 `A` |
| 重置游戏 | 点击「重置」按钮 或 按 `R` |
| 切换难度 | 点击「初级/中级/高级」按钮 或 按 `1/2/3` |

### 运行测试

```bash
python tests/test_engine.py
python tests/test_single_point.py
python tests/test_constraint.py
python tests/test_probability.py
python tests/test_solver.py
```

## 求解表现

| 难度 | 规格 | 胜率 |
|------|------|:---:|
| 初级 | 9×9, 10 雷 | ~100% |
| 中级 | 16×16, 40 雷 | ~90% |
| 高级 | 16×30, 99 雷 | ~70% |

> 高级难度因雷密度高、约束关系稀疏，概率猜测占比增加，胜率依赖采样质量。
